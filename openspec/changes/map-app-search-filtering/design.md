# Design

## Search Semantics

搜索框先尝试匹配本地地图资产中的国家、地区和地点。命中时只执行地图视角变更，并尝试在当前查询结果里选中对应热点。该定位状态通过独立 chip 展示，不参与 SQL 查询。

未命中本地地名时，输入被视为全文搜索 `q`。全文搜索继续复用 `/api/hotspots` 现有能力，覆盖地区名、热点摘要、参与方、来源元数据和事件代码。

## Filters

主工作台的筛选状态只保留：

- `date`
- `channel`
- `q`

API 和数据库查询层仍保留 `region` 参数，用于调试、脚本或后续专门筛选入口，但主界面不再由搜索框写入它。

## Data Flow

地图标记使用当前视野 bbox 查询，排行榜不使用 bbox：

- map: `bbox + date + channel + q + sort`
- ranking: `date + channel + q + sort + limit`

这样拖动、缩放只影响地图上的可见点，不会把全文搜索或筛选后的排行榜裁剪到当前屏幕范围。

## Map Controls

地图控件由前端 SVG 地图的本地投影状态实现，不引入第三方地图 SDK 控件。放大/缩小围绕当前地图中心；回全球恢复默认视图；适配当前结果根据当前热点点位计算包围框。

## Declutter Copy

智能避让仍在前端完成。overlay 文案明确展示 `X/Y`，并在隐藏重叠点时提示放大查看，避免用户误以为数据丢失。
